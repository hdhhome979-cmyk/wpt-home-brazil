最该警惕 · 抢组织者
PPPoker / PokerBROS / Suprema
不是抢安装量，而是抢host、club owner和WhatsApp社群节点。Suprema在巴西本地化最深，"像本地生态的一部分"。FIVE必须把房主当一等公民，否则只能抢散客。
最该学习 · 边界正确
ClubGG / Pokerrrr 2 / EasyPoker
ClubGG是"品牌安全+Club能力"的最佳样本。Pokerrrr 2和EasyPoker证明朋友局工具不需要复杂，只需要够快够顺。FIVE要赢=品牌信任 + 比他们更省事。
不正面硬碰 · 不同战场
Zynga / WSOP / Governor / Poker Heat
金币大厅头部，靠品牌惯性和大池子吃休闲时长。跟FIVE不在同一个使用场景。不该学他们的大厅路线，但该学信任感和品牌清晰度。

关键阻塞项（4项）
① 回放显示问题（正在发酵） — "发两次牌"场景下回放分成两个链接、积分显示看似异常，导致玩家误解为发牌不公平。产品发牌机制本身没问题，但回放展示方式已引发KOL-A俱乐部玩家质疑，Ring Game已经组不起来。研发已确认短期先加标识，合并牌局改动较大。
② MTT无法收取活跃分 — 上线三个月未修复，修复预计4月下旬。直接阻塞巴西赛事合作推进。
③ MTT配置深度不够 — 目前仅100bb起始，大比赛需300bb。巴西线下赛事方有明确需求。
④ 缺少门票系统 — 卫星赛无法正常运作，只能口头说明奖励。
PR预案 — 4.7日Media-A PR上线取决于计分牌测试结果。如4.7前确认无重大bug→照常上线；如需等下版本修复→推迟至4月中下旬。需提前跟Media-A沟通时间弹性。

房主扩散双引擎 · 转介绍 + Founders Program
横向扩散 · Club Host转介绍4月启动
机制：老房主推荐新房主 → 验证14天（建议初期降为：14天内3天有局+每天30手）→ 发放奖励
激励方案：混合模式 — 少量USDT真钱（种子期标杆案例）+ 游戏内钻石 + 周边物件 + 荣誉称号体系
种子节点：从已有Founders Club（十几家）中挑3-5家最活跃的作为首批推荐人
SOP：4月上旬确定方案→4月中旬种子启动→14天验证→5月扩大到所有合作Club
预期效果：主要提升小型/新建房主的获取效率，且被推荐来的房主因有老房主背书，质量高于纯投放线索
纵向密度 · Founders Program已在运行
现状：巴西BD本地团队（Local-A/Local-B，向BD-C汇报）已推进十几家Club加入

① Open Home Game — 广告线索+小组织者，轻量onboarding，接转介绍体系

模型逻辑：五档分层。激活当月=首周邀请+3周×周均。之后每月=周均×4。邀请经漏斗后计入活跃。存量按月流失衰减。房主自身计入DAU。玩家→房主下月生效归入小型档。

转介绍+Founders对模型的影响：转介绍主要加速"小型/新建房主"的获取效率——被老房主推荐来的新Club质量高于纯投放线索（有信任背书，更容易达到有效标准）。Founders Program则通过区域密度建设提升中型Club房主的获取速度。两者共同降低有效房主获取成本，但当前模型中未单独建模（因激励方案和转化率均无实际数据），建议5月转介绍全面开放后根据实际数据补充修正。

局限：内容型KOL数据未跑出(w1=15为估算)。漏斗参数需数据校准。未考虑回放显示问题导致的信任流失、MTT功能缺失对赛事合作的阻塞等产品侧变量。

达标路径：核心靠中型(累计78)+小型(累计165)。KOL和大型Club是杠杆但量有限。转介绍和Founders是加速器——如果跑通，可降低对投放线索的依赖。如中小型拓展不及预期，需提升漏斗参数弥补。
